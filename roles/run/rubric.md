## Run Skill Rubric

### 检查清单（10 项，每项 5 分 = 50 分）

对每一项，达标给 5 分，不达标给 0 分。

- [ ] **计划可见**: 输出中有明确的计划阶段，包含任务理解、成功标准定义、子任务拆解
- [ ] **成功标准具体**: 成功标准是可验证的 checklist，每条标准能明确判断通过/未通过，不含模糊词汇
- [ ] **评估可见**: 输出中有明确的评估阶段，逐项对照成功标准进行检查
- [ ] **功能完整**: 任务要求的所有功能点都已实现（API 端点、UI 功能、代码结构等）
- [ ] **输入验证**: 所有外部输入（HTTP 参数、表单输入、函数参数）在处理前经过验证
- [ ] **错误处理**: 覆盖明显的错误路径（无效输入、资源不存在、边界情况），有具体的错误消息
- [ ] **代码可运行**: 生成的代码能编译/运行无语法错误，逻辑正确
- [ ] **项目结构清晰**: 代码组织合理，分层明确，文件命名规范
- [ ] **无过度设计**: 没有实现用户没有要求的功能，没有不必要的抽象层或配置
- [ ] **安全实践**: 没有硬编码密钥、SQL 注入、XSS 等明显安全问题（代码任务适用）；没有违反常识的设计缺陷

### 质量维度（5 个维度，每个 1-5 分，求和 × 2 = 10-50 分）

1. **计划质量** (1=计划笼统或缺失, 3=有计划但成功标准模糊, 5=计划精确、标准可验证、子任务粒度适中)
2. **执行完整性** (1=多处遗漏未实现, 3=主要功能完成但细节缺失, 5=所有要求完整实现，边界情况覆盖)
3. **评估严格度** (1=无评估或走过场, 3=有评估但标准放松, 5=严格逐项对照计划标准检查，有明确的通过/未通过判定)
4. **代码/输出质量** (1=代码混乱或逻辑错误, 3=可读但风格不统一, 5=代码整洁、符合语言惯例、注释恰当)
5. **迭代效率** (1=不必要的循环浪费, 3=循环次数合理但单次效率低, 5=最少迭代达到标准，无冗余步骤)
