# Harness Rubric

### Checklist (10 items, 5 points each = 50 points)

For each item, award 5 points if clearly present, 0 if missing or inadequate.

- [ ] **规则解析准确**: 正确理解规则的触发条件、约束行为和豁免条件，用自己的话复述并与用户意图一致
- [ ] **工具类型选择合理**: 根据决策树选择最合适的约束工具类型（hook/CLAUDE.md/settings/skill），并有明确的选择理由
- [ ] **约束工具格式正确**: hook 脚本（stdin JSON 解析/退出码 0=通过 2=阻止/stdout 格式）、CLAUDE.md（结构化规则含触发条件与行为）、skill（完整 frontmatter+流程）、settings（合法 JSON+权限语法），所选类型格式均符合规范
- [ ] **无冗余约束**: 不为同一事件/目标创建多个独立约束（hook 脚本按事件合并、CLAUDE.md 规则按主题合并、settings 条目按类型合并），避免重复检查
- [ ] **最小开销实现**: 硬拦截用 hook（唯一可靠手段，settings deny 在 bypass 模式无效）；软约束用 CLAUDE.md；减少弹窗用 settings allow；hook 脚本优先 sh、复杂用 node、执行 < 100ms；使用 matcher/if 精确过滤
- [ ] **安装验证**: 安装后验证文件权限（chmod +x）、JSON 格式（jq）、脚本可执行性
- [ ] **防御性编码与最小权限**: hook 脚本处理异常输入不因自身错误阻塞操作；修改 settings.json 增量更新不覆盖已有配置；约束只拦截必要操作，不过度阻止正常行为
- [ ] **可逆可删**: 每条约束可以单独删除或禁用，不影响其他约束的运行
- [ ] **汇报清晰**: 输出包含约束类型、文件路径、触发条件、约束行为、验证结果
- [ ] **实时生效**: 安装的约束工具不需要重启会话即可生效（利用 hooks 热加载或 CLAUDE.md 即时加载）

### Quality Dimensions (5 dimensions, scored 1-5, sum * 2 = 10-50 points)

1. **约束准确性** (1=约束与用户规则不一致，3=基本匹配但有偏差，5=精确匹配用户意图，无遗漏无多余)
2. **架构合理性** (1=约束工具选择不当或结构混乱，3=选择基本合理，5=每个约束都选择了最优工具类型，系统结构清晰)
3. **性能意识** (1=明显拖慢 CC 响应，3=有性能考虑但不彻底，5=最小化 hook 数量和执行时间，延迟增加 < 500ms)
4. **健壮性** (1=脚本崩溃或误改配置，3=处理常见边界，5=全面处理异常输入、并发、配置安全，settings.json 增量修改不丢配置)
5. **可维护性** (1=约束规则难以理解和修改，3=有注释和结构但不够清晰，5=每个约束独立、可定位、有文档，团队可轻松增删改)
